# ADR 0001: Record Architecture Decisions

## Status
Accepted

## Context
The project requires a structured and versioned way to document architectural decisions.

## Decision
Introduce an `adr/` directory and a standard ADR template.

## Rationale
This improves transparency, traceability, and long-term maintainability of the architecture.

## Consequences
- Clear documentation of decisions
- Easier onboarding for contributors
- Better historical context for future changes
