# Proactivity and Initiative Model

## Overview

The proactivity and initiative model determines when the agent should take initiative, propose next steps, or remain reactive. It balances helpfulness with user autonomy and safety.

---

## Types of Initiative

### Reactive Mode
Respond only to explicit user requests.

### Guided Mode
Offer structured next steps when beneficial.

### Proactive Mode
Anticipate needs in long-term workflows.

---

## Initiative Triggers

- multi-step tasks,
- stalled progress,
- recurring patterns,
- long-term goals,
- safety concerns,
- user preferences.

---

## Initiative Constraints

- never override user intent,
- avoid excessive proactivity,
- respect safety boundaries,
- align with memory-stored preferences.

---

## Initiative Workflow

1. Evaluate context and intent.  
2. Check initiative triggers.  
3. Apply constraints and safety rules.  
4. Decide initiative level.  
5. Generate appropriate suggestion or remain reactive.  

---

## Integration with Other Systems

- planning uses initiative to shape next steps,
- memory stores initiative preferences,
- safety restricts proactive suggestions,
- dialogue management ensures smooth flow.

---

## Design Principles

- respect for user autonomy,
- minimalism,
- safety,
- contextual awareness,
- predictability.

---

## Future Extensions

- adaptive initiative tuning,
- long-horizon initiative graphs,
- preference-based initiative profiles.
