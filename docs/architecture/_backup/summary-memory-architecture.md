# Summary Memory Architecture

## Overview

Summary memory stores stable, long‑term information distilled from repeated interactions. It represents durable knowledge about user preferences, long‑term goals, and recurring patterns. Unlike episodic memory, summary memory persists across sessions and provides the foundation for personalization and long‑horizon reasoning.

---

## Purpose of Summary Memory

### Preserve Long‑Term Knowledge
Stores:
- stable preferences,
- recurring goals,
- consistent behavioral patterns.

### Support Multi‑Session Continuity
Enables:
- seamless resumption of long projects,
- consistent tone and style,
- long‑term personalization.

### Improve Strategic Reasoning
Provides:
- context for long‑horizon planning,
- grounding for meta‑reasoning,
- guidance for adaptive strategies.

---

## Memory Structure

### Preference Entries
Examples:
- writing style,
- tone,
- formatting choices.

### Long‑Term Goals
Examples:
- ongoing projects,
- multi‑session tasks,
- recurring objectives.

### Stable Facts
Examples:
- non‑sensitive user information,
- durable constraints,
- confirmed patterns.

### Behavioral Patterns
Examples:
- typical workflows,
- common misunderstandings,
- repeated requests.

---

## Lifecycle

### Distillation
Information is added only when:
- repeated across sessions,
- stable over time,
- safe and non‑sensitive.

### Updating
Entries are updated when:
- patterns change,
- preferences evolve,
- contradictions appear.

### Pruning
Entries are removed when:
- outdated,
- irrelevant,
- unsafe.

---

## Integration with Other Systems

### Long‑Horizon Reasoning
Summary memory provides:
- continuity across sessions,
- stable goals,
- consistent preferences.

### Adaptive Planning
It influences:
- plan structure,
- strategy selection,
- tool usage patterns.

### Reflection
Reflection updates summary memory with:
- distilled insights,
- stable patterns,
- long‑term improvements.

---

## Design Principles

- **Stability** — only durable information is stored.
- **Safety** — no sensitive or personal data.
- **Minimalism** — only high‑value entries persist.
- **Determinism** — consistent updates across sessions.
- **Interpretability** — entries are human‑readable.
