# Code of Conduct  
**Agent AI Lab — Contributor Code of Conduct**  
Version 1.0.0

This Code of Conduct defines expected behavior for all contributors and participants in the Agent AI Lab project.  
Our goal is to maintain a respectful, safe, and collaborative environment that supports open research, responsible AI development, and high‑quality engineering practices.

---

## 1. Our Standards

We are committed to creating an environment where all participants:

- treat each other with respect and professionalism  
- collaborate constructively and transparently  
- provide helpful, actionable feedback  
- value diversity of thought, background, and expertise  
- prioritize safety, ethics, and responsible AI practices  
- maintain integrity in documentation, code, and governance  
- follow project guidelines, versioning rules, and architectural conventions  

Examples of **positive behavior**:

- using inclusive and respectful language  
- being open to different viewpoints  
- gracefully accepting constructive criticism  
- focusing on what is best for the project  
- supporting others and helping newcomers  
- documenting decisions clearly and transparently  

Examples of **unacceptable behavior**:

- harassment, discrimination, or personal attacks  
- trolling, insulting comments, or derogatory remarks  
- publishing private information without permission  
- introducing unsafe, misleading, or harmful AI patterns  
- intentionally breaking documentation structure or governance rules  
- submitting malicious or deceptive contributions  
- bypassing safety, ethics, or compliance guidelines  

---

## 2. Responsibilities of Maintainers

Project maintainers are responsible for:

- reviewing contributions fairly and consistently  
- enforcing this Code of Conduct  
- clarifying standards and expectations  
- moderating discussions and resolving conflicts  
- taking corrective action when inappropriate behavior occurs  
- ensuring architectural integrity and documentation quality  

Maintainers have the right to:

- edit, reject, or revert contributions that violate standards  
- remove comments or discussions that break the rules  
- temporarily or permanently ban contributors for misconduct  

---

## 3. Scope

This Code of Conduct applies to:

- all project spaces (GitHub issues, PRs, discussions, wiki, docs)  
- all interactions related to the project  
- public and private communication involving project contributors  
- contributions to documentation, code, diagrams, governance, or tooling  

---

## 4. Responsible AI Expectations

Because this project focuses on AI agent architecture, contributors must:

- avoid introducing unsafe or harmful agent behaviors  
- follow safety, ethics, and governance guidelines  
- document risks, limitations, and assumptions  
- avoid speculative claims without evidence  
- ensure transparency and traceability of architectural decisions  
- respect privacy, security, and compliance requirements  

Contributions that undermine safety or governance will be rejected.

---

## 5. Reporting Violations

If you observe or experience a violation of this Code of Conduct:

- report it privately via GitHub Security Advisories or direct maintainer contact  
- include relevant context, links, or evidence  
- do **not** open a public issue for sensitive matters  

Reports will be handled confidentially and respectfully.

---

## 6. Enforcement

Maintainers will investigate all reports and determine appropriate action, which may include:

- warnings  
- temporary or permanent bans  
- removal of contributions  
- escalation to GitHub Trust & Safety if necessary  

Maintainers must document enforcement decisions when appropriate.
